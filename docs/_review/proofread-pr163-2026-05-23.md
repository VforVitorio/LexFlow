## Cross-cutting checks

- **`prefers-reduced-motion`** — covered everywhere it matters:
  - `frontend/src/index.css:128-135` — global kill-switch for `*, *::before, *::after` (animation-duration 0.01ms, iteration 1, transition 0.01ms !important). This is a backstop so any forgotten landing animation still degrades.
  - `landing.css` — explicit `@media (prefers-reduced-motion: reduce)` blocks for: section-eyebrow dot (l.301), hero pill pulse-dot (l.408), feature hover (l.594), nav-underline (l.1737), stack-row (l.1775), lf-caret/lf-thinking/lf-chip-in (l.1829), lf-hint (l.1618), author cards (l.1572).
  - `RevealSection.tsx` — uses framer-motion's `useReducedMotion` to render a bare wrapper.
  - `HowItWorks.tsx` — uses `useReducedMotion` to render a static 3-card grid.
  - `ChatMockup.tsx` — local `prefersReducedMotion()` matchMedia check jumps to final state.
  - `index.css:189-194` — view-transition + theme-fade collapse to 0.01ms.
  No gaps found.

- **Token discipline (#161 follow-up)** — every animation introduced in this PR uses `--ease-out` + `--dur-*`. The exceptions worth listing are pre-existing or intentionally infinite breathing animations:
  1. `landing.css:34` `.landing-root` root: `transition: background-color 200ms ease, color 200ms ease` (pre-existing).
  2. `landing.css:129,157,183,677,692,962,1312,1339,1430` — legacy `120ms` transitions on nav-links, icon-btn, seg, feature-link, feature-art-link, lang/theme buttons. Pre-existing; not introduced by this PR.
  3. `landing.css:1460-1464,1536-1540` — author-card / author-link transitions hardcoded to `200ms` / `180ms cubic-bezier(0.2,0,0,1)`. Introduced in the previous PR (6ebf1b0 "Authors section"); still uses the right curve but inlined the literal instead of the token. **Follow-up: 2 sites, swap to `var(--dur-micro) var(--ease-out)`.**
  4. Infinite "breathing" animations (`lf-pulse-dot 1.8s`, `lf-hero-shimmer 9s`, `lf-thinking-dot 1.1s`, `lf-caret-blink 0.95s`, `lf-node-breath 4s`, `lf-edge-flow 6s`, `phase-status glow`) intentionally do NOT come from the four discrete duration tokens — these are continuous loops, not one-shot UI transitions, so token discipline does not apply.
  Total flagged off-token sites in NEW code: 2 (author-card, author-link). Threshold for follow-up issue was 4+, so no new issue required.

- **Wiring gaps** — none found:
  - `.nav-underline` class is declared in CSS and rendered in `Nav.tsx`; inline style writes `transform` + `width` driven by `active` state which updates from the IntersectionObserver.
  - `.pulse-dot`, `.hiw-rail*`, `.hiw-anchor`, `.lf-caret`, `.lf-thinking`, `.lf-chip-in`, `.lf-term-*`, `.lf-hint`, `.feature-art-wrap` are all referenced from .tsx files (verified via grep).
  - `ChatMockup` is rendered from `Layers.tsx` at `idx === 2`.
  - `HowItWorks` is mounted in `LandingPage.tsx` and deliberately NOT wrapped in `RevealSection` (the wrapper would distort the framer-motion `useScroll` offsets).
  - `TerminalBlock` rendered in `BuiltOn.tsx`; the `terminal` i18n array is passed unchanged.
  - Cloudflare beacon placeholder is replaced in BOTH `dist/index.html` AND `dist/404.html` (the SPA fallback).
  Minor observation: `nav-spacer` exists alongside the `nav-links` `margin-left: 16px` — both work, no clash.

- **i18n parity** — `frontend/src/i18n/locales/en/landing.json` and `es/landing.json` have IDENTICAL key sets (recursive diff returned empty in both directions). `howItWorks` exists in both with `eyebrow`, `title`, `sub`, and `scenes` (3 items × `title`+`body`). No drift.

## Recommendation

Merge as-is. Every one of the 13 issues lands at or above spec — #156 is the only one I would call "partial" and only because the issue body floated an optional second scope (a new detail-panel section) that the PR explicitly skipped while still delivering the row treatment named in the title; that is a defensible scoping call, not a miss. Reduced-motion coverage is comprehensive (one global backstop + nine explicit per-feature overrides + three JS-side checks). Token usage on new code is clean; the only follow-up worth tracking is swapping the two `author-*` transitions (introduced one PR earlier) onto the `--dur-*`/`--ease-out` tokens for consistency — too small for a blocker. The i18n parity is exact and the Cloudflare beacon workflow is gracefully no-op when the secret is unset. Ship it; open a tiny `chore(landing)` follow-up for the two author-card transitions.
